# BioModelBench — VEP multi-eval v0

## Task

Predict a single `score` in [0, 1] for each variant in `test.parquet`.
Higher = more likely to be a **functionally impactful variant**.

`test.parquet` is a mixed test set. Some of the variants are drawn from
one biological question (GWAS-fine-mapped causal SNPs for common
complex traits) and others from a different biological question (clinical
pathogenic vs. benign classification). **You are not told which is which.**
The grader has the per-source labels and will report AUPRC / AUROC
separately for each hidden partition.

The point of the task is not to reverse-engineer which variant is from
which source. The point is to build **one model whose ranking is good
across both questions** — a model of "variant importance" that transfers.

Test labels are hidden. You cannot evaluate against them.

## What's shipped

### `test.parquet` — 41,400 rows (approximate; regenerated by build.py)

Columns: `chrom` (string, 1-22), `pos` (int, hg38 1-based), `ref`, `alt`.
**No `label`. No features. No source flag.**

## Where to look for training labels

### Allowed sources

- **Open Targets Genetics** — fine-mapped GWAS credible sets
  (`https://ftp.ebi.ac.uk/pub/databases/opentargets/platform/latest/output/credible_set/`).
  Filter to `studyType == "gwas"`, positives are variants with
  `posteriorProbability >= 0.9` in a credible set.
- **CausalDB** — smaller curated fine-mapping database.
- **gnomAD** — constraint scores (LOEUF, missense z, s_het), allele
  frequency. Safe as features; can serve as weak labels for functional
  impact.
- **UCSC bigWig / bigBed hub** — conservation (phyloP, phastCons, GERP,
  Zoonomia phyloP-241m), chromatin state, DNase, ATAC, TF ChIP,
  histone marks.
- **ENCODE cCREs** — regulatory-element annotations.
- **Ensembl VEP REST API** — variant consequence, TSS distance, regulatory
  feature overlap.
- **CADD API** — `https://cadd.gs.washington.edu/api/v1.7`, POST/GET
  per-variant.
- **HuggingFace model weights** — any model repo (Enformer, Borzoi,
  Nucleotide-Transformer, HyenaDNA, Evo 2, ESM-2, …).
- **hg38 reference FASTA** — `https://hgdownload.soe.ucsc.edu/goldenPath/hg38/bigZips/hg38.fa.gz`.

### Blacklisted sources — DO NOT FETCH

- **TraitGym-derivative:** `huggingface.co/datasets/songlab/TraitGym/*`
  (`complex_traits_matched_9/*`, `complex_traits_all/*`,
  `complex_traits_v22_matched_9/*`, `mendelian_traits_*`, `AUPRC/*`
  prediction tables).
- **Kanai et al. 2022 UKBB fine-mapping** (Zenodo, Finucane lab).
- **PolyFun / SuSiE UKBB outputs** for TraitGym's 62 UKBB traits.
- **NCBI ClinVar releases** — `ftp.ncbi.nlm.nih.gov/pub/clinvar/*` in any
  form (VCF, TSV, XML), and downstream aggregations that expose ClinVar
  labels (e.g., AlphaMissense's public prediction table, which was
  trained on ClinVar and would collapse the eval; VEP's ClinVar plugin
  outputs; MyVariant.info variant-level ClinVar significance).
- Any Google Drive / Dropbox / Zenodo mirror of the above.

If a URL looks like it might expose the labels for either hidden test
partition, stop and use an allowed source instead. Log every fetch in
`training_manifest.json`.

### Anti-leak protocol (mandatory)

1. Whatever training labels you assemble, **drop every row whose
   `(chrom, pos, ref, alt)` appears in `test.parquet`.**
2. Prefer training sources that are demonstrably independent of both
   TraitGym's UK Biobank fine-mapping and ClinVar's clinical annotations.
3. Report your anti-leak drop count in `method.md`.

Modal A10G, 8 CPU, 32 GB RAM, 24 GB VRAM. `torch / transformers / peft /
xgboost / lightgbm / scikit-learn / pandas / pyarrow / numpy / pyBigWig /
pysam / biopython / requests` pre-installed. `pip install <x>` works.
Wall clock ~10 hours.

## Deliverables

- `answer.parquet` — columns `chrom, pos, ref, alt, score`, one row per
  test variant, `score ∈ [0, 1]`.
- `method.md` (< 500 words) — training source(s) chosen, features
  extracted, model + validation strategy, anti-leak drop count.
- `training_manifest.json` — every URL / API / dataset touched.

## Reference bars

Reference AUPRCs on similar splits (approximate — actual bars depend on
the exact subsample the build script produced):

- **TraitGym partition** — base rate ~10%. phyloP-241m single feature
  ≈ 0.235. Published supervised methods 0.35–0.45.
- **ClinVar partition** — base rate ~50% (P/LP vs. B/LB). Single-feature
  conservation ≈ 0.70–0.80. AlphaMissense / REVEL / ClinPred sit around
  0.95+ on this slice (but were trained on ClinVar, so are near-circular).

A model that scores well on one partition but not the other is a
domain-specific model. A model that scores modestly on both captures a
transferable signal. Both are valid findings.

## Rules recap

- No test labels visible. No self-evaluation against test.
- No fetching from blacklisted sources.
- Anti-leak filter is your responsibility.
- One model, one submission. No test-source detection tricks.
